# Contributing & forking

budget-tracker is open source under the [MIT License](LICENSE). Here's how it's meant to be used and
worked on.

## Forks are welcome 🍴

You are free — and encouraged — to **fork this project** and make it your own:

- Run your own private, self-hosted instance.
- Modify, extend, or rip out anything you like for your personal use.
- Learn from the code or reuse parts of it in your own projects.

The MIT License grants all of this. No permission needed — just keep the copyright
notice and license text in copies, as the license requires.

## We are not accepting direct contributions

This is a personal project published for others to use and adapt, **not** a community-maintained
codebase. To keep it simple and focused, we **do not accept pull requests, patches, or feature
contributions** into this repository.

Practically, that means:

- **Pull requests will not be merged.** Please don't open one expecting it to be reviewed or accepted.
- **No roadmap or feature requests.** The direction is driven by the maintainer's own needs.
- **No support guarantees.** Issues may be read but are not guaranteed a response, triage, or fix.

If you want a change, **make it in your own fork** — that's exactly what forks are for.

## Found a security issue?

If you discover something genuinely security-sensitive (e.g. a way to leak another user's data),
please report it privately to the maintainer rather than opening a public issue. There is no bug
bounty; it's simply the courteous way to handle it.

## In short

> **Fork it, change it, run it yourself — go for it.** Just don't expect changes to be pulled back
> upstream here.
